## 3.3 Requisitos funcionales y no funcionales

### 3.3.1 Requisitos Funcionales

| ID      | Descripción                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|---------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| CRRF-001 | El sistema debe verificar la disponibilidad de los consultores en tiempo real, validando que no existan conflictos de horario al momento de realizar una reserva. Para ello, se debe comprobar que el intervalo de tiempo seleccionado no se cruce con otras sesiones confirmadas. En caso de conflicto, la reserva debe ser rechazada y el sistema deberá sugerir horarios alternativos.                                                                                                                                                   |
| CRRF-002 | El sistema debe procesar las reservas de sesiones mediante un flujo transaccional que incluya validación de disponibilidad, confirmación de datos y bloqueo temporal del horario seleccionado. Si el proceso no se completa dentro de un tiempo límite definido, el sistema debe liberar automáticamente el horario para evitar bloqueos indebidos.                                                                                                                                                                                         |
| CRRF-003 | El sistema debe gestionar la comunicación en tiempo real entre usuarios y consultores mediante un módulo de mensajería. Debe permitir el envío y la recepción de mensajes instantáneos, garantizando su entrega y almacenamiento.                                                                                                                                                                                                                                                                                                           |
| CRRF-004 | El sistema debe registrar y almacenar el historial completo de mensajes asociados a cada sesión de asesoría. Cada mensaje debe incluir información del emisor, receptor, fecha, hora y contenido, permitiendo su consulta posterior de manera ordenada y consistente.                                                                                                                                                                                                                                                                       |
| CRRF-005 | El sistema debe calcular automáticamente la calificación promedio de un consultor a partir de todas las valoraciones recibidas. Este cálculo debe actualizarse en tiempo real cada vez que se registre una nueva calificación y debe reflejarse inmediatamente en el perfil del consultor.                                                                                                                                                                                                                                                  |
| CRRF-006 | El sistema debe priorizar la visibilidad de consultores con plan premium en los resultados de búsqueda. Para ello, debe aplicar un algoritmo de ordenamiento que considere factores como suscripción activa, calificación y relevancia, garantizando que los perfiles premium aparezcan destacados sin excluir otros resultados relevantes.                                                                                                                                                                                                 |
| CRRF-007 | El sistema debe gestionar el ciclo de vida de una sesión de asesoría mediante estados definidos (pendiente, confirmada, en curso, completada, cancelada). Cada transición de estado debe estar sujeta a validaciones específicas y debe ser registrada para permitir su trazabilidad en cualquier momento.                                                                                                                                                                                                                                  |
| CRF-001 | El sistema debe permitir al usuario realizar búsquedas de especialistas por categoría, experiencia y calificación. Los resultados deben actualizarse dinámicamente según los filtros aplicados.                                                  |
| CRF-002 | El sistema debe permitir al usuario visualizar el perfil detallado de un consultor. Si el consultor no existe, la operación debe ser rechazada mostrando un mensaje adecuado.                                                                    |
| CRF-003 | El sistema debe permitir al usuario cancelar una sesión programada. Si la cancelación se realiza fuera del tiempo permitido, el sistema debe aplicar las políticas correspondientes definidas por la plataforma.                                 |
| CRF-004 | El sistema debe permitir al usuario consultar sus sesiones programadas, mostrando información como fecha, hora, estado y consultor asociado. Esta información debe actualizarse automáticamente ante cualquier cambio.                           |
| CRF-005 | El sistema debe permitir al usuario calificar una sesión únicamente si esta se encuentra en estado “completada”. La calificación debe incluir una puntuación y un comentario opcional.                                                           |
| CRF-006 | El sistema debe permitir al consultor definir, modificar y eliminar su disponibilidad horaria. Si existen sesiones asociadas, el sistema debe impedir cambios que generen conflictos de horario.                                                 |
| CRF-007 | El sistema debe permitir al consultor consultar sus sesiones agendadas, incluyendo información del usuario, fecha, hora y estado de cada sesión.                                                                                                 |
| CRF-008 | El sistema debe permitir al usuario consultar su historial de asesorías realizadas. Para cada sesión, se debe mostrar información como fecha, consultor asociado, estado de la sesión, calificación otorgada y cualquier observación registrada. |
| CRF-009 | El sistema debe permitir consultar la lista de especialistas disponibles, mostrando información resumida como nombre, especialidad, calificación promedio y tarifa por sesión.                                                                   |
| CRF-010 | El sistema debe permitir al usuario ordenar y filtrar la lista de especialistas según criterios como precio, calificación, experiencia o disponibilidad, facilitando la toma de decisiones.                                                      |
| CRF-011 | El sistema debe permitir gestionar categorías de especialización utilizadas para clasificar a los consultores. Estas categorías deben poder ser utilizadas en los procesos de búsqueda y filtrado dentro de la plataforma.                       |
| IRF-001 | El sistema debe permitir el registro de nuevos usuarios mediante correo electrónico, contraseña y nombre de usuario . El correo debe cumplir con un formato válido, no exceder los 255 caracteres, ser único en el sistema y almacenarse en minúsculas. La contraseña debe tener entre 8 y 128 caracteres e incluir al menos una letra minúscula y un dígito; el uso de letras mayúsculas y caracteres especiales será opcional. Si no se especifica un rol durante el registro, el sistema debe asignar automáticamente el rol de usuario. |
| IRF-002 | El sistema debe permitir la autenticación de usuarios registrados mediante correo electrónico y contraseña. Debe validar que el usuario exista y que la contraseña coincida con la almacenada mediante una comparación segura de hash. En caso de credenciales inválidas, el acceso debe ser rechazado. Si la autenticación es exitosa, el sistema debe generar un token de acceso y un token de actualización asociados al usuario.                                                                                                        |
| IRF-003 | El sistema debe verificar la validez de los tokens de acceso utilizados en las solicitudes. Esta verificación debe incluir comprobaciones de expiración, integridad y origen, determinando si la sesión asociada es válida o no.                                                                                                                                                                                                                                                                                                            |
| IRF-004 | El sistema debe proporcionar al usuario autenticado acceso a su información básica, incluyendo identificador, correo electrónico y roles asignados, siempre que presente credenciales válidas.                                                                                                                                                                                                                                                                                                                                              |
| IRF-005 | El sistema debe permitir la modificación de las credenciales del usuario, limitándose al cambio de contraseña y nombre de usuario. Antes de aplicar los cambios, se debe confirmar la identidad del usuario mediante un mecanismo de validación.                                                                                                                                                                                                                                                                                            |
| PRF-001 | El sistema debe permitir la creación de un perfil de usuario asociado a una cuenta registrada. El perfil debe incluir información básica como nombre, apellido y una imagen opcional. Para usuarios con rol de consultor, el sistema debe habilitar campos adicionales como especialidades, descripción profesional y tarifa por sesión.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| PRF-002 | El sistema debe permitir consultar el perfil de un usuario mediante su identificador único. La información devuelta debe incluir datos personales básicos y, en el caso de consultores, información profesional y configuración de servicios ofrecidos.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| PRF-003 | El sistema debe permitir a los consultores actualizar su perfil profesional, incluyendo especialidades, experiencia, descripción y tarifas. Los cambios realizados deben reflejarse inmediatamente en el perfil visible para otros usuarios.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| PRF-004 | El sistema debe calcular y mantener la calificación promedio de cada consultor a partir de las valoraciones recibidas en sesiones completadas. Este valor debe actualizarse automáticamente cada vez que se registre una nueva calificación.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| PRF-005 | El sistema debe permitir consultar el historial de asesorías de un usuario, mostrando para cada sesión información como fecha, consultor asociado, estado y resultado. Este historial debe estar disponible tanto para usuarios como para consultores según corresponda.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| PRF-006 | El sistema debe permitir a los consultores acceder a métricas de desempeño, incluyendo número de sesiones realizadas, tasa de sesiones completadas, ingresos generados y calificación promedio. Estas métricas deben calcularse a partir de los datos históricos registrados.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| PRF-007 | El sistema debe registrar las valoraciones y comentarios asociados a cada sesión, vinculándolos tanto al usuario como al consultor. Esta información debe ser utilizada para construir la reputación del consultor dentro de la plataforma.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| PRF-008 | El sistema debe permitir visualizar las valoraciones de un consultor en su perfil público, mostrando la calificación promedio y los comentarios realizados por otros usuarios, garantizando la transparencia del servicio.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| RF-001  | El sistema debe permitir al consultor crear una publicación informativa dentro de su perfil, especificando título, descripción y categoría del contenido (por ejemplo: consejos, artículos, recursos). Una vez creada, la publicación debe estar disponible para los usuarios que visiten su perfil.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| RF-002  | El sistema debe permitir al consultor modificar una publicación previamente creada, actualizando su contenido, título o categoría. El sistema debe registrar la fecha de modificación para fines de seguimiento.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| RF-003  | El sistema debe permitir al consultor eliminar una publicación de su perfil. Al eliminarla, el contenido debe dejar de estar disponible tanto para él como para los usuarios.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| RF-004  | El sistema debe permitir al consultor adjuntar archivos a una publicación (imágenes o documentos), respetando límites de cantidad y tamaño definidos por la plataforma. Los archivos deben almacenarse de forma segura.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| RF-005  | El sistema debe permitir al usuario visualizar el perfil de un consultor junto con sus publicaciones, ordenadas cronológicamente.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| RF-006  | El sistema debe permitir al usuario acceder al detalle completo de una publicación, incluyendo contenido y archivos adjuntos si existen.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| RF-007  | El sistema debe permitir al usuario enviar consultas o mensajes relacionados a una publicación o servicio ofrecido por el consultor.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| RF-008  | El sistema debe permitir al usuario visualizar el estado de sus sesiones (pendiente, confirmada, completada o cancelada).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| RF-008  | El sistema debe permitir al usuario acceder al detalle de una sesión específica, incluyendo información del consultor, fecha, hora y estado.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| RF-010  | El sistema debe permitir al consultor visualizar las solicitudes de asesoría recibidas, mostrando información del usuario y estado de la solicitud.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| RF-011  | El sistema debe permitir al administrador gestionar los reportes recibidos, pudiendo revisar, resolver y tomar acciones sobre los casos reportados.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |

### 3.2.2 Requisitos no funcionales
| ID | Descripción |
| :--- | :--- |
| RNF-001 | El sistema debe responder a las solicitudes de búsqueda de especialistas en un tiempo máximo de 2 segundos bajo condiciones normales de carga (hasta 150 usuarios concurrentes). |
| RNF-002 | El sistema debe procesar la creación y confirmación de reservas de sesiones en un tiempo máximo de 3 segundos, incluyendo validación de disponibilidad y persistencia de datos. |
| RNF-003 | El sistema debe soportar al menos 300 usuarios concurrentes realizando operaciones de búsqueda, reserva y mensajería sin degradación significativa del rendimiento (tiempo de respuesta máximo de 4 segundos). |
| RNF-004 | El sistema debe estar disponible el 99.5% del tiempo mensual, excluyendo ventanas de mantenimiento programadas previamente notificadas. |
| RNF-005 | El sistema debe garantizar la integridad de las operaciones críticas (reservas, pagos, cancelaciones) mediante transacciones ACID en la base de datos. |
| RNF-006 | El sistema debe transmitir toda la información mediante el protocolo HTTPS con TLS 1.2 o superior, asegurando la confidencialidad de los datos en tránsito. |
| RNF-007 | El sistema debe cifrar las contraseñas de los usuarios utilizando algoritmos seguros (por ejemplo, BCrypt) antes de almacenarlas en la base de datos. |
| RNF-008 | El sistema debe validar todas las entradas de usuario, rechazando datos inválidos con respuestas HTTP 400, para prevenir errores y posibles ataques de inyección. |
| RNF-009 | El sistema debe registrar logs de las operaciones críticas (reservas, autenticación, pagos, mensajería) incluyendo timestamp, tipo de operación y resultado, utilizando niveles de logging (INFO, WARN, ERROR). |
| RNF-010 | El sistema debe permitir el almacenamiento persistente de datos en una base de datos relacional (PostgreSQL o equivalente), garantizando consistencia y disponibilidad. |
| RNF-011 | El sistema debe escalar horizontalmente mediante la implementación de microservicios desplegados en contenedores, permitiendo agregar nuevas instancias sin afectar la disponibilidad del sistema. |
| RNF-012 | El sistema debe procesar el envío y recepción de mensajes en tiempo real con una latencia máxima de 500 ms en condiciones normales de red. |
| RNF-013 | El sistema debe documentar todos los endpoints de la API mediante OpenAPI 3.0 (Swagger), proporcionando información clara sobre parámetros, respuestas y errores. |
| RNF-014 | El sistema debe manejar correctamente los errores, retornando códigos HTTP adecuados (200, 201, 400, 401, 404, 500) junto con mensajes descriptivos. |
| RNF-015 | El sistema debe generar identificadores únicos para entidades principales (usuarios, sesiones, reservas) utilizando UUID versión 4 para garantizar unicidad global. |
| RNF-016 | El sistema debe permitir la configuración de parámetros críticos (puertos, credenciales, límites de concurrencia) mediante variables de entorno, facilitando su despliegue en distintos entornos. |
| RNF-017 | El sistema debe mantener tiempos de carga de perfiles de consultores menores a 1.5 segundos bajo carga normal. |
| RNF-018 | El sistema debe implementar control de acceso basado en roles (usuario, consultor, administrador), validando permisos en cada operación protegida. |
| RNF-019 | El sistema debe registrar auditoría de cambios relevantes (modificación de perfil, cancelación de sesiones, valoraciones), incluyendo fecha, usuario y acción realizada. |
| RNF-020 | El sistema debe ser compatible con despliegues en contenedores Docker, permitiendo su ejecución en entornos de orquestación como Kubernetes. |
